## Section Guidelines

### Executive Summary
- **Length:** 1-2 sentences maximum
- **Tone:** Clear, direct, executive-level
- **Content:** Overall health + key highlight + critical alert (if any)
- **Status Emojis:** ğŸ“— Green (Good/Excellent), ğŸ“™ Yellow (Fair), ğŸ“• Red (Needs Attention)

### Business Health Dashboard
- **Table Format:** Use for quick visual scanning
- **Trends:** Compare to last week (â†—ï¸ up, â†’ stable, â†˜ï¸ down)
- **Scores:** Always out of 100 for consistency
- **Status:** Text + emoji for quick recognition

### Financial Performance
- **Currency:** Always include $ symbol and use commas for thousands
- **Percentages:** Always show progress vs target
- **Alerts:** Use âš ï¸ for overdue invoices, budget overruns
- **Notable Transactions:** Only include if >$500 or unusual

### Operational Performance
- **Metrics:** Focus on efficiency (cycle time, response time, completion rate)
- **Alerts:** Flag anything impacting client satisfaction
- **Trends:** Compare to expected/target values

### Social Media Performance
- **Multi-Platform:** Always break down by platform
- **Engagement Rate:** More important than absolute numbers
- **Top Content:** Show what's working for content strategy insights

### Goal Progress
- **One Goal Per Block:** Clear, scannable format
- **Status Indicators:** ğŸŸ¢ Green (on track), ğŸŸ¡ Yellow (at risk), ğŸ”´ Red (behind)
- **Actionable:** Include blockers and next milestones

### Bottlenecks
- **Prioritize:** Critical first, then high, then medium
- **Quantify:** Always show expected vs actual
- **Root Cause:** Don't just identify, explain why
- **Omit if Empty:** If no bottlenecks, say so positively

### Proactive Recommendations
- **Specific:** Actionable, not vague
- **Quantified:** Include savings, benefits, effort
- **Prioritized:** Help user decide what to act on first
- **Realistic:** Don't recommend what can't be done

### Action Items
- **Checkboxes:** Use Markdown task format for user to check off
- **Prioritized:** Critical, High, Medium
- **Specific:** Clear what needs to be done
- **Realistic:** User can complete in reasonable time

## Best Practices

1. **Be Consistent:** Use the same format every week for easy comparison
2. **Be Concise:** Executive summary should be scannable in 30 seconds
3. **Be Honest:** Don't sugarcoat issues - flag problems clearly
4. **Be Actionable:** Every recommendation should have clear next steps
5. **Be Data-Driven:** Support insights with numbers
6. **Be Forward-Looking:** Include upcoming items, not just retrospective
7. **Be Prioritized:** Use priority levels to guide decision-making
8. **Be Visual:** Use emojis, tables, and formatting for quick scanning
